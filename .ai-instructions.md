## Current Issue: npm ci Failure (Job #76664200066)

### Problem
The CI/CD pipeline is failing at the `npm ci` step due to a mismatch between `package.json` and `package-lock.json`.

**Error Details:**
- `package.json` specifies `esbuild@^0.25.0`
- `package-lock.json` contains `esbuild@0.25.12` with all platform binaries
- The lock file is missing entries for `esbuild@0.28.0` and its platform-specific binaries

### Root Cause
Someone likely updated `package.json` but forgot to run `npm install` to regenerate `package-lock.json` before committing.

### Solution

**Option 1: Update lock file (Recommended)**
```bash
npm install
git add package.json package-lock.json
git commit -m "Update lock file to match package.json dependencies"
git push
```

**Option 2: Revert package.json to match lock file**
If you want to keep `esbuild@0.25.x`:
```bash
# Edit package.json and ensure:
"esbuild": "^0.25.0",

# Then run:
npm install
git add package.json package-lock.json
git commit -m "Revert esbuild to 0.25.x and update lock file"
git push
```

### Verification
After applying the fix, the CI pipeline should:
1. Successfully complete `npm ci` without errors
2. Pass all subsequent build and test steps
